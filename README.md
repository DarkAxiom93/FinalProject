# 🎲 Virtual Casino System

**מערכת קזינו וירטואלית מתקדמת בשפת C**

פרויקט גמר המדמה קזינו שלם הרץ בטרמינל (Console UI), עם חמישה משחקים, מערכת חשבונות מאובטחת, אנטי-צ'יט, טבלת שיאים ופאנל ניהול נסתר. הפרויקט בנוי בארכיטקטורה מודולרית: לוגיקת המשחקים, מנוע התצוגה (Graphics), מערכת החשבונות (Account) והתשתיות (Utils) מופרדים לחלוטין זה מזה.

---

## ✨ עיקרי הפרויקט

- **ארכיטקטורה מודולרית** — כל משחק הוא קובץ `.c`/`.h` עצמאי; כל הציורים וה-ASCII Art מרוכזים ב-`graphics.c` בלבד.
- **אנטי-צ'יט** — קבצי השמירה (`data/<name>.bin`) נשמרים עם XOR encoding וחתימת Checksum מתמטית. שינוי ידני של הקובץ גורם לזיהוי אוטומטי וטיפול בחשבון כ"חשוד".
- **מנוע קלפים משותף** (`cards.c`) — בניית וערבוב (Fisher–Yates) של חפיסות, בשימוש משותף ע"י הפוקר (חפיסה אחת) והבלאק-ג'ק (Shoe של 6 חפיסות).
- **PRNG כפול** — מנוע Xoroshiro128+ (`casino_rand`) לתוצאות המשחק בפועל, ומנוע נפרד ומבודד (`visual_rand`) לאנימציות בלבד, כך שאנימציה לא יכולה להשפיע על התוצאה האמיתית.
- **קלט מאובטח** — כל קלט עובר טיהור (`get_safe_int`, `is_valid_name`) המונע Buffer Overflow, Path Traversal וקריסות.
- **ניהול זיכרון** — הקצאות דינמיות (`malloc`/`realloc`) משוחררות (`free`) באופן עקבי בסיום כל סיבוב/משחק.
- **חשבון בנק נפרד** — `bank_balance` מוגן בנפרד מה-`balance` השוטף, כדי שלא ניתן להפסיד את כל הכסף בבת אחת.

## 🎰 המשחקים

| # | משחק | תיאור |
|---|------|-------|
| 1 | **Roulette** | רולטה מלאה עם הימורים מורכבים (Straight, Split, Street, Corner, Color/Dozens), ביטול הימור (Undo) והיסטוריית סיבובים אחרונים. |
| 2 | **VIP Blackjack** | Shoe של 6 חפיסות, תמיכה ב-Split וב-Double Down, ודילר עם לוגיקת Hit on Soft 17. |
| 3 | **Ultimate Texas Hold'em** | פוקר נגד הדילר עם הימורי Ante / Blind / Trips, וזיהוי ידיים מ-High Card ועד Straight Flush. |
| 4 | **Slot Machine** | מכונת מזל עם אנימציית סיבוב חיה בתוך מסגרת ASCII קבועה, הימור רציף (Spin Again) ומערכת Jackpot. |
| 5 | **Winner Sportsbook** | טופס טוטו דינמי — מגריל 3 משחקים ויחסי זכייה (Odds) ממאגר קבוצות, ומדמה תוצאות אמיתיות. |

מעבר למשחקים, יש גם **Cashier** (הפקדות/משיכות מהבנק), **Leaderboard** (Hall of Fame) ו-**My Stats** (סטטיסטיקת רווח/הפסד לכל הקריירה).

## 🔒 פאנל ניהול (Admin Panel)

המערכת כוללת פאנל ניהול נסתר למפתחים/הנהלת הקזינו, עם שתי שכבות הגנה:

1. **קוד כניסה** — מוזרק כקבוע preprocessor (`ADMIN_ENTRY_CODE`) בהגדרות הקומפילציה של ה-Build (ולא מופיע כטקסט גלוי בקוד המקור), ומוקש בתפריט הראשי לצד שאר האפשרויות.
2. **סיסמת מנהל** — נבדקת מול hash (`hash_password` / `get_admin_hash`) ולא נשמרת בקוד כטקסט רגיל.

הקוד והסיסמה בפועל אינם מתועדים כאן בכוונה — הם פרטי תצורה פרטיים של הבנייה (Build), לא תיעוד ציבורי.

אפשרויות הניהול כוללות: הענקת בונוס לשחקן הנוכחי, ריגול (Audit) מאובטח על נתוני שחקנים אחרים, החרמת כספים, חסימת/שחרור שחקנים, ודוח רווח כללי של הבית (House Profit Report).

## ⚙️ קימפול והרצה (Build & Run)

הפרויקט פותח ונבדק ב-**Visual Studio 2022** עבור פלטפורמת **x64**, ומשתמש ב-API ייחודי ל-Windows (`<Windows.h>`, `_getch()`, `system("cls")`).

1. פתחו את `Casino.sln` ב-Visual Studio.
2. בחרו קונפיגורציית **Debug** או **Release** בפלטפורמת **x64**.
3. הריצו (Build → Build Solution, ואז Debug → Start Without Debugging).

תיקיית `data/` להיצרת קבצי השמירה נוצרת אוטומטית בריצה הראשונה.

> הפרויקט משתמש ב-`#define _CRT_SECURE_NO_WARNINGS` כדי לאפשר שימוש בפונקציות C קלאסיות (`scanf`, `strcpy`).

### בדיקות (Tests)

קיים מערך טסטים עצמאי (`tests.c`) שבודק את לוגיקת הכסף והמשחקים בלי תלות ב-`main()`. להרצה מתוך **x64 Native Tools Command Prompt for VS**, בתיקיית `Casino\Casino`:

```bat
cl /W3 /Fe:..\tests.exe tests.c account.c blackjack.c poker.c roulette.c ^
   cards.c utils.c graphics.c football.c slots.c leaderboard.c admin.c

..\tests.exe
```
